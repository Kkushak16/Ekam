Ekam Documentation: Developer Setup & API Guides

Welcome to the official developer documentation for Ekam. This guide helps you configure your development environment, connect to transactional databases, run migration schemas, and start the local chat server.

1. Quick Start Installation

To run Ekam locally, ensure you have Node.js (version 18 or above) and npm installed. Clone the repository and install core dependencies in the root directory:

cd Ekam
npm install
cd frontend
npm install

2. Database Schemas Configuration

Ekam relies on a dual-database design to support heavy user traffic and protect transactional integrity. Set up your local databases using the instructions below:

A. PostgreSQL Setup via Supabase

Configure user profiles, usernames, room relations, and friendship associations inside PostgreSQL. Apply the SQL schema migrations located in the migration folder:

-- Example Username migration
ALTER TABLE profiles ADD COLUMN username VARCHAR(255) UNIQUE;

B. MongoDB Setup

All high-throughput messaging content is routed and stored inside a MongoDB collection. Make sure you set your connection URL string in the root environment configuration file:

MONGODB_URI=mongodb://localhost:27017/ekam_db

3. Running the Server

Start the nodemon backend developer server. The server runs on port 5000 by default and handles WebSocket, SSE, and HTTP REST routes:

npm run dev

To launch the interactive frontend Vite application, run:

npm run dev --prefix frontend

For additional API routes and details about specific WebPush notification channels, refer to the files under the root services and routes folders.
